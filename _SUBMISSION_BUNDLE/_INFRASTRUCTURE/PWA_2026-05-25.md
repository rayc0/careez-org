## Service Worker Strategy

```
Cache version:  careez-v1.1-2026-05-25
Install:        Pre-caches shell assets (/, /index.html, all main pages, icons, og-image)
Activate:       Deletes old cache versions; claims all clients immediately
Fetch routing:
  - /api/*           → skip (network-only, no SW intercept)
  - Cross-origin     → skip (Cloudflare Analytics, fonts, etc.)
  - Non-GET          → skip
  - navigate mode    → network-first → cache → /offline.html
  - static assets    → cache-first → network (update cache on hit)
```

## Cache Version Policy

To invalidate the cache on next deploy, update the constant in `sw.js`:

```js
const CACHE_VERSION = 'careez-v1.1-2026-05-25';
```

Increment minor version (e.g. `v1.2`) or append date. Old caches are purged automatically on `activate`.
